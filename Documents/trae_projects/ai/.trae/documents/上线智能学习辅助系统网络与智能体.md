# 上线智能学习辅助系统网络与智能体

## 目标
成功启动OpenAgents网络服务和智能学习辅助系统的三个核心智能体，确保系统可以正常使用。

## 步骤

### 1. 检查当前系统状态
- 停止所有相关进程（网络服务和智能体）
- 检查端口占用情况（8700, 8600）
- 验证环境配置

### 2. 启动OpenAgents网络服务
- 使用 `openagents network start ./learning_assistant_network` 启动网络
- 验证网络健康状态：`http://localhost:8700/api/health`
- 确认HTTP和gRPC服务都已启动

### 3. 启动智能体
- 启动学情分析Agent：`python my_first_network/agents/learning_analysis_agent.py`
- 启动资源匹配Agent：`python my_first_network/agents/resource_matching_agent.py`
- 启动练习优化Agent：`python my_first_network/agents/exercise_optimization_agent.py`

### 4. 验证系统功能
- 检查智能体连接状态
- 运行测试脚本验证核心功能
- 测试网络与智能体的通信

### 5. 提供使用指导
- 如何在OpenAgents Studio中连接
- 核心功能的使用方法
- 常见问题排查

## 预期结果
- 网络服务稳定运行
- 三个智能体成功连接
- 系统核心功能正常
- 可以通过OpenAgents Studio访问

## 技术细节
- **网络地址**：localhost:8700
- **LLM服务**：已配置CloseAI API
- **智能体数量**：3个核心智能体
- **通信协议**：gRPC + HTTP

## 风险与应对
- 端口占用：更换端口或停止占用进程
- 连接失败：检查网络配置和智能体代码
- LLM服务异常：切换到本地Ollama备用配置
