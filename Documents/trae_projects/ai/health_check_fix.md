# 健康检查失败解决方案

## 问题原因
健康检查失败是因为 OpenAgents 平台无法访问您的本地网络：

1. **网络隔离**：OpenAgents 平台在云端运行，无法直接访问您本地机器上的 `localhost` 或私有 IP 地址
2. **防火墙限制**：您的网络环境可能有防火墙或 NAT 配置，阻止外部访问
3. **端口映射**：本地端口 8700 只能在您的机器内部访问

## 解决方案

### 选项 1：使用公网隧道（推荐）
我们尝试了 localtunnel，但遇到了连接问题。建议使用 ngrok：

1. 下载并安装 ngrok：[https://ngrok.com/download](https://ngrok.com/download)
2. 注册一个免费账户
3. 运行以下命令创建隧道：
   ```bash
   ngrok http 8700
   ```
4. 获取生成的公网 URL（例如：`https://xxxx.ngrok.io`）
5. 在发布表单中使用该 URL 的主机名和端口

### 选项 2：部署到云服务器
1. 将您的网络部署到具有公网 IP 的云服务器（如 AWS、阿里云等）
2. 确保服务器的 8700 端口已开放
3. 在发布表单中使用云服务器的公网 IP 和端口

### 选项 3：本地开发模式
如果您只是在本地开发和测试，可以使用以下方法：

1. 保持当前配置不变
2. 在发布表单中继续使用 `localhost` 和端口 `8700`
3. 虽然健康检查会失败，但您仍然可以完成发布流程
4. 注意：这样发布的网络只能在您的本地机器上访问
